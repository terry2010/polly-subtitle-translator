## 一、规划原则

1. **不和 Apple 系统功能正面竞争**：macOS 27 已覆盖"实时播放英语视频出中文字幕"场景，二期不投入这个方向
2. **不和 SmartSub 正面竞争**：SmartSub（4K stars，开源，Tauri 技术栈）已在"语音转写 + 多引擎翻译 + 批量处理"做到极致，二期不追这条路
3. **聚焦竞品盲区**：在线字幕搜索、内嵌流提取、播放预览、Jellyfin/Emby 插件、NAS 服务化——SmartSub 和 Apple 都不做
4. **先巩固再扩展**：先补齐一期遗留（macOS 适配、时间码编辑），再开新方向
5. **语音转写大幅降级**：SmartSub 已有 6 大 ASR 模型 + GPU 加速 + 4K stars，追不上也没必要追，P3 降为 P4 观望
5. **可独立交付**：每个 P1/P2 功能应能独立交付，不依赖其他 P1/P2 先完成

## 二、优先级分级标准

| 级别 | 含义 | 判断标准 |
| --- | --- | --- |
| P0 | 必须先做 | 一期遗留的阻塞项，不做会影响二期功能展开 |
| P1 | 核心护城河 | Apple 永远不会做的领域，直接决定项目竞争力 |
| P2 | 生态扩展 | 扩大用户触达面，提升产品粘性 |
| P3 | 辅助能力 | 有差异化价值但受众较小，资源允许时做 |
| P4 | 体验打磨 | 锦上添花，不紧急 |

## 三、功能清单与优先级

### P0：补齐一期遗留

这些是一期已标记"暂未实现"或"仅 Windows"的项，是二期展开的基础。

| # | 功能 | 说明 | 预估工作量 | 依赖 |
| --- | --- | --- | --- | --- |
| P0-1 | macOS 适配 | FFmpeg/libmpv 下载改为跨平台，右键菜单改 Service，keyring 改 Keychain，自动更新适配 macOS | 大 | 无 |
| P0-2 | 字幕时间码编辑 | 表格编辑器支持时间码（开始/结束）直接编辑 | 小 | 无 |
| P0-3 | 时间轴偏移快捷键 | ±0.1s / ±1s 步进快捷键 | 小 | P0-2 |
| P0-4 | 复制行 | 表格编辑器复制整行 | 小 | 无 |

**P0 完成标准**：macOS 可编译运行 + 编辑器功能补齐。

---

### P1：核心护城河

这些是 SmartSub 和 Apple 系统功能都不会碰的领域，是项目的核心竞争力。

| # | 功能 | 说明 | 预估工作量 | 依赖 | 与竞品关系 |
| --- | --- | --- | --- | --- | --- |
| P1-1 | 在线字幕搜索强化 | 扩展搜索源、优化关键词简化、自动匹配视频、搜索结果质量排序 | 中 | 无 | SmartSub 不做，核心差异 |
| P1-2 | 批量处理 | 一次导入多个视频/字幕文件，队列处理（提取→翻译→合并），进度总览 | 大 | P0-1 | SmartSub 已有，不作为核心卖点 |
| P1-3 | Watch 模式 | 监控指定文件夹，新视频自动进入处理队列，处理完自动通知 | 中 | P1-2 | SmartSub 有文件夹扫描，差异化不大 |
| P1-4 | 翻译引擎扩展 | 增加 DeepL/Claude/GPT，至少追到 5-6 个引擎（SmartSub 有 17 个，追不平） | 中 | 无 | 缩小差距但不正面竞争 |
| P1-5 | 字幕质量工具 | 术语表、上下文翻译、翻译后校对界面 | 中 | P1-4 | SmartSub 有校对界面，需差异化 |
| P1-6 | 嵌回 mkv 语言标签优化 | 合并时正确设置 `language=chi` 等标签，确保 Jellyfin/Emby/Infuse 识别 | 小 | 无 | SmartSub 不关注此细节 |
| P1-7 | Apple TV/Infuse 链路优化 | 输出到 SMB 共享、Infuse 字幕流自动识别、端到端文档 | 小 | P0-1 | SmartSub 完全不做，核心差异 |

**P1 完成标准**：项目具备"在线搜字幕 → 提取 → 翻译 → 合并 → Apple TV 播放"的完整差异化链路。

### P2：生态扩展

扩大用户触达面，让项目融入 NAS/媒体服务器生态。**这是 SmartSub 完全不做的领域，是最大差异化机会。**

| # | 功能 | 说明 | 预估工作量 | 依赖 | 与竞品关系 |
| --- | --- | --- | --- | --- | --- |
| P2-1 | NAS/Docker 服务版 | 无头运行（无 GUI），HTTP API 触发处理，watch 文件夹自动处理，Docker 镜像分发 | 大 | P1-2, P1-3 | SmartSub 是桌面 app，不做服务化 |
| P2-2 | Jellyfin 插件 | GPL 开源插件，服务端自动提取→翻译→嵌回，对 Infuse 透明 | 中 | P2-1 | SmartSub 不做，核心护城河 |
| P2-3 | Emby 插件 | 同 Jellyfin，适配 Emby 插件框架 | 中 | P2-1 | SmartSub 不做，核心护城河 |
| P2-4 | HTTP 触发接口 | REST API，支持远程触发翻译任务（配合 Shortcut/自动化） | 小 | P2-1 | SmartSub 不做 |
| P2-5 | SMB/网络路径直接处理 | 支持直接打开 SMB/NFS 路径的视频处理，输出写回网络位置 | 中 | P0-1 | SmartSub 不做 |

**P2 完成标准**：项目可在 NAS 上无头运行，Jellyfin/Emby 用户无感使用。

## 五、实施阶段划分

### 阶段一：巩固基础（P0）

**目标**：macOS 可用 + 编辑器补齐

| 任务 | 优先级 |
| --- | --- |
| P0-1 macOS 适配 | 最高 |
| P0-2 时间码编辑 | 高 |
| P0-3 偏移快捷键 | 高 |
| P0-4 复制行 | 高 |

**交付物**：macOS + Windows 双平台可用的完整 v1 功能。

### 阶段二：核心护城河（P1）

**目标**：建立与 SmartSub 的差异化，强化"已有字幕处理 + 在线搜索 + Apple TV 链路"

| 任务 | 优先级 | 与 SmartSub 差异 |
| --- | --- | --- |
| P1-1 在线字幕搜索强化 | 最高 | SmartSub 不做 |
| P1-7 Apple TV/Infuse 链路优化 | 最高 | SmartSub 不做 |
| P1-6 语言标签优化 | 高 | SmartSub 不关注 |
| P1-4 翻译引擎扩展（加 DeepL/Claude） | 高 | 缩小差距（SmartSub 17 个，至少追到 5-6 个） |
| P1-2 批量处理 | 中 | SmartSub 已有，不作为核心卖点但需要 |
| P1-3 Watch 模式 | 中 | SmartSub 有文件夹扫描，差异化不大 |
| P1-5 字幕质量工具 | 中 | SmartSub 有校对，需差异化 |

**交付物**：在线搜索 → 提取 → 翻译 → 合并 → Apple TV 播放的完整差异化链路 + 批量处理基础能力。

### 阶段三：生态扩展（P2）

**目标**：融入 NAS/媒体服务器生态

| 任务 | 优先级 |
| --- | --- |
| P2-1 Docker 服务版 | 最高 |
| P2-5 SMB/网络路径处理 | 高 |
| P2-4 HTTP 触发接口 | 中 |
| P2-2 Jellyfin 插件 | 中 |
| P2-3 Emby 插件 | 中 |

**交付物**：Docker 镜像可部署在 NAS 上，watch 文件夹自动处理，Jellyfin/Emby 用户无感使用。

### 阶段四：语音转写（P3，建议放弃）

**启动条件**：~~P1 完成 + macOS 27 普及 + 用户反馈确认需求~~

**更新结论**：SmartSub 已有 6 大 ASR 模型 + GPU 加速 + 4K stars + 开源免费，本项目在语音转写领域无法形成竞争力。建议放弃自研，如用户有需求可推荐使用 SmartSub。

| 任务 | 状态 |
| --- | --- |
| ~~P3-1 离线转写~~ | 放弃 |
| ~~P3-2 转写编辑~~ | 放弃 |
| ~~P3-3 转写+翻译~~ | 放弃 |

**替代方案**：在文档/帮助中引导用户"无字幕视频用 SmartSub 转写，有字幕视频用 AI-SubTrans 翻译"，生态互补。

### 阶段五：体验打磨（P4，穿插进行）

**目标**：提升整体体验质量

P4 任务彼此独立，可在任何阶段穿插进行，不阻塞主线。

## 六、与 macOS 27 的关系标注

### 二期功能 vs macOS 27 覆盖范围

```
macOS 27 已覆盖（二期不做）：
  ❌ 实时播放英语视频 → 自动中文字幕
  ❌ 英语语音实时转写
  ❌ 英语字幕实时翻译

macOS 27 盲区（二期重点）：
  ✅ 在线字幕搜索强化          → P1-1
  ✅ 批量处理整个剧集库        → P1-2
  ✅ Watch 文件夹自动处理      → P1-3
  ✅ 非英语视频翻译            → P1-4
  ✅ 字幕文件编辑/校对/术语表  → P1-5
  ✅ 嵌回 mkv 跨设备播放       → P1-6
  ✅ Apple TV/Infuse 链路      → P1-7
  ✅ NAS/Docker 无头服务       → P2-1
  ✅ Jellyfin/Emby 服务端翻译  → P2-2/P2-3
  ✅ SMB/网络路径直接处理      → P2-5
```

### 动态调整机制

| 触发条件 | 调整动作 |
| --- | --- |
| macOS 28 扩展支持非英语源转写 | P3 语音转写彻底放弃（SmartSub + macOS 双重覆盖） |
| tvOS 28 支持语言翻译 | 强化 P1-7 Apple TV 链路（Infuse 用自定义播放器，仍需文件处理） |
| Apple 翻译质量显著提升 | P1-4 增加 LLM 引擎（Claude/GPT）保持质量优势 |
| macOS 27 生成字幕支持输出文件 | P3 语音转写彻底放弃 |

## 七、竞品影响与优先级调整

> 详见 `docs/phase2-competitor-analysis.md`

### 竞品格局

```
macOS 27（Apple 系统功能）：
  覆盖：实时播放英语视频出中文字幕
  不覆盖：文件处理、批量、跨设备、非英语源

SmartSub（4K stars，开源，Tauri）：
  覆盖：语音转写（6 ASR 模型）、17 翻译引擎、批量、GPU 加速、跨平台
  不覆盖：在线字幕搜索、内嵌流提取、播放预览、Jellyfin/Emby 插件、NAS 服务化

GeekLink（闭源，付费，仅 Apple Silicon）：
  覆盖：类似 SmartSub 但闭源付费
  市场表现：失败（几乎无下载量），可忽略

AI-SubTrans（本项目）：
  已有：内嵌流提取、3 翻译引擎、播放预览、在线搜索（3 源）、合并回 mkv
  差异化机会：在线搜索 + 播放预览 + Jellyfin/Emby 插件 + NAS 服务化 + Apple TV 链路
```

### 优先级调整总结

| 原规划项 | 调整后 | 原因 |
| --- | --- | --- |
| 在线字幕搜索 | ⬆️ **P1 最高优先** | SmartSub 完全不做，核心差异化 |
| Apple TV/Infuse 链路 | ⬆️ **新增 P1** | SmartSub 不做，核心差异化 |
| Jellyfin/Emby 插件 | ⬆️ **P2 最高优先** | SmartSub 不做，核心护城河 |
| NAS/Docker 服务版 | ⬆️ **P2 最高优先** | SmartSub 是桌面 app，不做服务化 |
| 批量处理 | ⬇️ P1 中优先 | SmartSub 已做到极致，不作为核心卖点 |
| 翻译引擎扩展 | ⬇️ P1 高但追不平 | SmartSub 17 个，追到 5-6 个够用 |
| 语音转写 | ⬇️ **放弃** | SmartSub 6 ASR + GPU + 4K stars，追不上 |

### 核心战略转变

```
原战略：做"语音转写 + 翻译 + 批量"的全能工具
        → 和 SmartSub 正面竞争，没有赢面

新战略：做"已有字幕处理 + 在线搜索 + 生态集成"的差异化工具
        → SmartSub 不做的领域，建立护城河

定位转变：
  原：AI 字幕提取翻译工具（和 SmartSub/GeekLink 同质）
  新：字幕搜索 + 提取 + 翻译 + 合并 + Apple TV 交付的完整链路工具
      面向"视频已有字幕但需要翻译"的用户
      而非"视频无字幕需要生成"的用户（那是 SmartSub 的领域）
```

### 生态互补策略

```
用户工作流：
  视频无字幕 → SmartSub 语音转写 → 得到 srt
  视频有字幕但没中文 → AI-SubTrans 提取 → 翻译 → 合并
  找不到字幕 → AI-SubTrans 在线搜索 → 下载 → 翻译
  想在 Apple TV 看 → AI-SubTrans 合并回 mkv → Infuse 播放
  NAS 自动化 → AI-SubTrans Docker + Jellyfin 插件
```

**不和 SmartSub 竞争，和 SmartSub 互补。** 用户可以同时用两个工具：SmartSub 处理无字幕视频，AI-SubTrans 处理有字幕视频 + 在线搜索 + Apple TV 交付。
